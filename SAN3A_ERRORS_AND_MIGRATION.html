1️⃣ ملخص الموقف الحالي

الموضوع: المشروع بدأ بـ MongoDB + Mongoose وتمت محاولة التحويل إلى PostgreSQL + Prisma.

الـ Prisma schema مكتوب بشكل كامل وجيد مع migrations فعلية في قاعدة البيانات. الـ Controllers الجديدة (authController, requestController, serviceController) تستخدم Prisma بشكل صحيح.

⚠️ المشكلة: بعض الملفات (adminController, dashboardController, seed.js, وأجزاء من userRoutes.js) لا تزال تستخدم Mongoose مع استدعاءات لملفات Models غير موجودة.

✅ الحل: إعادة كتابة هذه الملفات لاستخدام Prisma بدلاً من Mongoose، مع تعديلات بسيطة في الـ Frontend لتتوافق مع هيكل PostgreSQL.

2️⃣ جميع الأخطاء المكتشفة (12 خطأ)

خطأ فادح ❌ ملف adminController.js لا يزال يستخدم Mongoose بالكامل
📁 backend/src/controllers/adminController.js

الملف يستورد من ../models/userModel و ../models/requestModel — وهذه الملفات غير موجودة (لا يوجد مجلد models/ في src/). يستخدم دوال Mongoose مثل:

  • User.countDocuments()
  • User.findByIdAndUpdate()
  • User.find().select().sort().skip().limit()
  • Request.aggregate()
  • Request.find().populate()
  • request.save()
  • الحقول _id، $regex، $or، $match، $group، $lookup

السيرفر سيتعطل فور محاولة استخدام أي endpoint من admin routes.

✅ الحل: إعادة كتابة الملف بالكامل باستخدام Prisma (أنظر القسم 5)

يجب استبدال كل استدعاءات Mongoose باستعلامات Prisma المكافئة. تم كتابة الكود الكامل في الأسفل.

خطأ فادح ❌ ملف dashboardController.js لا يزال يستخدم Mongoose بالكامل
📁 backend/src/controllers/dashboardController.js

نفس المشكلة: يستورد من ../models/userModel و ../models/requestModel (غير موجودة).

  • يستخدم User.findByIdAndUpdate()
  • يستخدم req.user._id — في Prisma الـ id هو req.user.id
  • يستخدم Request.find().populate()
  • يستخدم Request.aggregate()
  • يستخدم $nin، $in، $match، $group
  • الحقل pricing.totalAmount غير موجود في Prisma (الاسم هو totalAmount فقط)
✅ الحل: إعادة كتابة الملف بالكامل باستخدام Prisma (أنظر القسم 6)
خطأ فادح ❌ ملف userRoutes.js يحتوي على Routes تستخدم Mongoose مباشرة
📁 backend/src/routes/userRoutes.js

الـ Routes التالية تستخدم require('../models/userModel') و require('../models/requestModel') (غير موجودة):

  • GET /public/:id — يستخدم User.findById(), Request.countDocuments()
  • GET /craftsmen — يستخدم User.find()
  • PATCH /update-profile — يستخدم User.findByIdAndUpdate(), req.user._id
✅ الحل: إعادة كتابة الـ Routes الثلاثة باستخدام Prisma (أنظر القسم 7)
خطأ فادح ❌ ملف seed.js بالكامل يستخدم Mongoose
📁 backend/seed.js

يستخدم:

  • mongoose.connect() — يستخدم process.env.MONGO_URI
  • Models من ./src/models/ (غير موجودة)
  • Service.updateOne(), Service.find()
  • User.deleteMany(), User.insertMany()
  • Request.deleteMany(), Request.insertMany()
  • حقل pricing (كـ subdocument) — غير موجود في Prisma
  • حقل location (كـ subdocument مع coordinates) — غير موجود في Prisma
  • تاريخ الطلبات يستخدم _id من MongoDB
✅ الحل: إعادة كتابة seed.js بالكامل باستخدام Prisma (أنظر القسم 4)
خطأ فادح ❌ requestRoutes.js يشير إلى دالة confirmBooking غير موجودة
📁 backend/src/routes/requestRoutes.js

السطر: router.post('/:requestId/confirm-booking', requestController.confirmBooking);

دالة confirmBooking غير معرفة في requestController.js — أي طلب لهذا الـ endpoint سيتسبب في TypeError: requestController.confirmBooking is not a function

✅ الحل: إضافة دالة confirmBooking إلى requestController.js (أنظر القسم 8)
متوسط ⚠️ Prisma Schema لا يتضمن حالة SELECTED, ACCEPTED, ARRIVED, DISPUTED, REFUNDED
📁 backend/prisma/schema.prisma

الـ Enum RequestStatus في الـ Schema يحتوي فقط على: PENDING_MATCHING, MATCHED, ON_THE_WAY, IN_PROGRESS, COMPLETED, CANCELLED

لكن قاعدة البيانات الفعلية (بعد الـ Migration الثاني) تحتوي على الإضافات: SELECTED, ACCEPTED, ARRIVED, DISPUTED, REFUNDED

النتيجة: الـ Schema غير متزامن مع الـ Database — أمر prisma generate سينتج Client لا يعرف هذه القيم.

✅ الحل: تحديث الـ Enum في schema.prisma (أنظر القسم 3)
متوسط ⚠️ Frontend يستخدم _id بدلاً من id
📁 frontend/src/app/(auth)/register/page.tsx — السطر: localStorage.setItem('user_id', response.data.data?.user?._id)

في MongoDB، الـ ID هو _id. في PostgreSQL/Prisma، الـ ID هو id. هذا السطر سيخزّن undefined في localStorage.

أيضاً في Map.tsx وملفات Dashboard: يتم استخدام _id في الـ fallback data والـ API responses.

✅ الحل: تغيير _id إلى id في كل مكان (أنظر القسم 9)
متوسط ⚠️ Frontend يستخدم pricing.totalAmount بدلاً من totalAmount
📁 جميع ملفات frontend/src/app/dashboard/craftsman/*.tsx

في MongoDB، كان السعر في subdocument pricing.totalAmount. في Prisma Schema، الحقل هو totalAmount مباشرة (بدون pricing.).

الـ Dashboard الخاص بالـ craftsman يستخدم job.pricing?.totalAmount — سيظهر undefined أو 0.

✅ الحل: تغيير job.pricing?.totalAmount إلى job.totalAmount (أنظر القسم 9)
متوسط ⚠️ Frontend يستخدم location.coordinates بدلاً من latitude/longitude
📁 frontend/src/components/Map.tsx — السطر: const raw = c.location?.coordinates

في MongoDB، الموقع كان في location.coordinates (GeoJSON array). في Prisma، الإحداثيات في حقول منفصلة: latitude و longitude.

✅ الحل: تعديل Map.tsx لاستخدام c.latitude و c.longitude (أنظر القسم 9)
بسيط ℹ️ تضارب بين JWT Cookie و localStorage
📁 backend/src/controllers/authController.js + frontend/src/middleware.ts

الـ Backend يضع JWT في Cookie (res.cookie("jwt", token, ...)) لكنه يقبل أيضاً Authorization: Bearer header.

الـ Frontend middleware يتحقق من Cookie فقط. بينما صفحات Login/Register تخزن الـ token في localStorage.

هذا ليس خطأ فادحاً لكنه يسبب عدم اتساق — أحياناً يعمل وأحياناً لا.

✅ الحل: توحيد الطريقة — إما Cookie فقط أو localStorage فقط (يُنصح باستمرار استخدام Cookie لأنه أكثر أمناً مع httpOnly)
بسيط ℹ️ دور role في Prisma Enum vs String
📁 backend/prisma/schema.prisma

Prisma schema يعرّف الـ role كـ Enum UserRole بقيم CUSTOMER, CRAFTSMAN, ADMIN (Capital letters).

الـ Controllers تستخدم 'customer', 'craftsman', 'admin' (lowercase). Prisma يتطلب القيمة المطابقة للـ Enum.

لذا في authController.js يجب تمرير 'CUSTOMER' بدلاً من 'customer'.

✅ الحل: تغيير القيم إلى uppercase في كل مكان: 'CUSTOMER', 'CRAFTSMAN', 'ADMIN'
بسيط ℹ️ ملفات appError.js و catchAsync.js فارغة
📁 backend/src/utils/appError.js, backend/src/utils/catchAsync.js

كلا الملفين فارغان. قد يستخدمها أحد في المستقبل أو قد تكون بقايا من الـ MongoDB code القديم.

يمكن حذفها أو ملؤها لاحقاً — لكن لا تؤثر على تشغيل المشروع حالياً.

✅ الحل: حذف الملفين أو تركهما — لا يؤثران على التشغيل.

function getStatusHistory(finalStatus) {
  const base = ['PENDING_MATCHING'];
  switch (finalStatus) {
    case 'IN_PROGRESS': return [...base, 'SELECTED', 'ACCEPTED', 'ARRIVED', 'IN_PROGRESS'];
    case 'COMPLETED': return [...base, 'SELECTED', 'ACCEPTED', 'IN_PROGRESS', 'COMPLETED'];
    case 'DISPUTED': return [...base, 'SELECTED', 'ACCEPTED', 'IN_PROGRESS', 'COMPLETED', 'DISPUTED'];
    case 'PENDING_MATCHING': return base;
    default: return base;
  }
}

const prisma = require('../utils/prisma');

exports.getDashboardStats = async (req, res) => {
  try {
    const [
      totalUsers,
      activeWorkers,
      revenueResult,
      openDisputes,
      requestsByStatus,
      recentRequests,
      serviceDistribution,
      weeklyRevenue,
    ] = await Promise.all([
      prisma.user.count({ where: { role: { not: 'ADMIN' } } }),
      prisma.user.count({ where: { role: 'CRAFTSMAN', isAvailable: true } }),
      prisma.request.aggregate({
        where: { status: 'COMPLETED', isPaid: true },
        _sum: { totalAmount: true },
      }),
      prisma.request.count({ where: { status: 'DISPUTED' } }),
      prisma.request.groupBy({
        by: ['status'],
        _count: { id: true },
      }),
      prisma.request.findMany({
        take: 5,
        orderBy: { createdAt: 'desc' },
        include: {
          client: { select: { id: true, name: true } },
          craftsman: { select: { id: true, name: true } },
          service: { select: { nameAr: true } },
        },
      }),
      prisma.request.groupBy({
        by: ['serviceId'],
        _count: { id: true },
      }),
      prisma.request.groupBy({
        by: ['createdAt'],
        where: { status: 'COMPLETED', isPaid: true },
        _sum: { totalAmount: true },
      }),
    ]);

    const totalRevenue = revenueResult._sum?.totalAmount || 0;

    const statusMap = {};
    requestsByStatus.forEach((r) => { statusMap[r.status] = r._count.id; });

    // Get service names for distribution
    const serviceIds = serviceDistribution.map(s => s.serviceId);
    const services = await prisma.service.findMany({
      where: { id: { in: serviceIds } },
      select: { id: true, nameAr: true },
    });
    const serviceNameMap = {};
    services.forEach(s => { serviceNameMap[s.id] = s.nameAr; });
    const distribution = serviceDistribution.map(s => ({
      name: serviceNameMap[s.serviceId] || 'Unknown',
      count: s._count.id,
    }));

    res.status(200).json({
      status: 'success',
      data: {
        stats: {
          totalUsers,
          activeWorkers,
          todayRevenue: totalRevenue,
          openDisputes: openDisputes || 0,
          totalRequests: Object.values(statusMap).reduce((a, b) => a + b, 0),
        },
        requestsByStatus: statusMap,
        recentRequests,
        serviceDistribution: distribution,
        weeklyRevenue,
      },
    });
  } catch (err) {
    res.status(500).json({ status: 'error', message: err.message });
  }
};

exports.getUsers = async (req, res) => {
  try {
    const { search, role, status, page = 1, limit = 10 } = req.query;
    const where = {};

    if (role && role !== 'all') {
      where.role = role === 'customer' ? 'CUSTOMER'
        : role === 'craftsman' ? 'CRAFTSMAN'
        : role === 'admin' ? 'ADMIN' : role;
    }
    if (status === 'active') where.isActive = true;
    else if (status === 'inactive') where.isActive = false;

    if (search) {
      where.OR = [
        { name: { contains: search, mode: 'insensitive' } },
        { email: { contains: search, mode: 'insensitive' } },
        { phone: { contains: search, mode: 'insensitive' } },
      ];
    }

    const skip = (parseInt(page) - 1) * parseInt(limit);
    const [users, total] = await Promise.all([
      prisma.user.findMany({
        where,
        orderBy: { createdAt: 'desc' },
        skip,
        take: parseInt(limit),
      }),
      prisma.user.count({ where }),
    ]);

    res.status(200).json({
      status: 'success',
      results: users.length,
      data: {
        users,
        pagination: {
          currentPage: parseInt(page),
          totalPages: Math.ceil(total / parseInt(limit)),
          totalItems: total,
        },
      },
    });
  } catch (err) {
    res.status(500).json({ status: 'error', message: err.message });
  }
};

exports.getRequests = async (req, res) => {

    const [requests, total] = await Promise.all([
      prisma.request.findMany({
        where,
        include: {
          client: { select: { id: true, name: true, phone: true, email: true } },
          craftsman: { select: { id: true, name: true, phone: true } },
          service: { select: { nameAr: true, nameEn: true } },
        },
        orderBy: { createdAt: 'desc' },
        skip,
        take: parseInt(limit),
      }),
      prisma.request.count({ where }),
    ]);

    res.status(200).json({
      status: 'success',
      results: requests.length,
      data: {
        requests,
        pagination: {
          currentPage: parseInt(page),
          totalPages: Math.ceil(total / parseInt(limit)),
          totalItems: total,
        },
      },
    });
  } catch (err) {
    res.status(500).json({ status: 'error', message: err.message });
  }
};

exports.resolveDispute = async (req, res) => {
  try {
    const { resolution, note } = req.body;
    const request = await prisma.request.findUnique({ where: { id: req.params.id } });

    if (!request) {
      return res.status(404).json({ status: 'fail', message: 'الطلب غير موجود' });
    }

    const newStatus = resolution === 'refund' ? 'REFUNDED' : 'COMPLETED';
    await prisma.request.update({
      where: { id: req.params.id },
      data: {
        status: newStatus,
        statusHistory: {
          create: {
            status: newStatus,
            changedAt: new Date(),
            note: `تم حل النزاع: ${note || ''}`,
          },
        },
      },
    });

    res.status(200).json({ status: 'success', data: { request } });
  } catch (err) {
    res.status(500).json({ status: 'error', message: err.message });
  }
};
